# Minimal fixture template

Use this template for tiny non-body support artifacts that accompany a generated feature or payload.

## Required structure

- keep the file as small as possible
- match the target file format already used by the repo
- end with exactly one trailing newline for text files

## Authoring rules

- Only create the fixture when it is explicitly required by the approved plan.
- Prefer existing repo conventions over invented structure.
- If the artifact format is unclear or unsupported, stop with `needs-review` instead of improvising.
