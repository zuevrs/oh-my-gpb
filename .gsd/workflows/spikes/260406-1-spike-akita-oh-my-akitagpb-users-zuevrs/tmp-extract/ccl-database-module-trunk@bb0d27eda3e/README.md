# ccl-database-module
## Назначение сервиса
Сервис ccl-database-module предназначен для взаимодействия с базами данных через выполнение SQL-запросов и их обработку. Он включает классы для подключения к БД и реализации Cucumber-шагов. Сервис относится к домену автоматизации тестирования баз данных.
# Описание процесса
В сервисе реализован следующий бизнес-процесс: Управление доступом к БД:
1. Настройка подключения к БД через файл конфигурации (database.conf).
2. Выполнение именованных SQL-запросов из файлов в директории /resources/queries.
3. Сохранение и проверка результатов запросов в переменные.
4. Выполнение операций обновления, удаления и добавления данных с проверкой ответа.

## Конфигурация
Для возможности взаимодействия с БД с помощью данного модуля необходимо в .conf-файле в ресурсах проекта (обычно это отдельный файл database.conf)
указать параметры подключения к БД следующим образом (для каждой БД - отдельный набор параметров):
```
application {
    driver = "org.postgresql.Driver"
    url = "jdbc:postgresql://ccl-v0101pgl:5432/ccl_application"
    username = "tuz_username"
    password = "tuz_password"
    queryFilePath = "application.sql"
}
```
В поле "queryFilePath" - указывается имя файла, находящегося в директории _**/resources/queries**_, в котором хранятся именованные запросы к
соответствующей БД в следующем формате:
```
-- найти Camunda процесс по name
select *
from dictionaries.process_definition
where name = '${process_name}';
```
Тогда в тесте можно будет использовать шаги данного модуля в следующем виде:
```
И в БД "application" выполняем запрос "найти Camunda процесс по name" с параметрами и сохраняем результат в "dbResponse"
```

## Список шагов модуля
```
@И("^в БД \"(.+)\" выполняем запрос \"(.+)\" с параметрами$")
@И("^в БД \"(.+)\" выполняем запрос \"(.+)\" с параметрами из таблицы( без перезаписи переменных)?$")
@И("^в БД \"(.+)\" выполняем запрос \"(.+)\" с параметрами и сохраняем результат в \"(.+)\"$")
@И("^в БД \"(.+)\" выполняем запрос \"(.+)\" с параметрами из таблицы( без перезаписи переменных)? и сохраняем результат в \"(.+)\"$")
@И("^в БД \"(.+)\" выполняем запрос \"(.+)\" и получаем пустой ответ$")
@И("^в ответе от БД \"(.+)\", проверяем значения полей$")
@И("^в ответе от БД \"(.+)\" проверяем значения полей с учётом порядка строк:?$")
@И("^в ответе от БД \"(.+)\" проверяем значения полей без учёта порядка строк:?$")
@И("^из ответа от БД \"(.*)\" значение столбца \"(.*)\" сохранено как ?(текст|объект) в переменную \"(.*)\"$")
@И("^в БД \"(.+)\" выполняем запрос \"(.+)\" с параметрами и сохраняем результат в переменную \"(.+)\" в виде json строки$")
@И("^в БД \"(.+)\" выполняем запрос (?:обновления|удаления|добавления) \"(.+)\"$")
@И("^в БД \"(.+)\" выполняем запрос (?:обновления|удаления|добавления) \"(.+)\" с параметрами из таблицы$")
@И("^в БД \"(.+)\" выполняем запрос (?:обновления|удаления|добавления) \"(.+)\" и проверяем полученный ответ$")
@И("^в БД \"(.+)\" выполняем запрос (?:обновления|удаления|добавления) \"(.+)\" с параметрами из таблицы и проверяем полученный ответ$")
```

## Требования
Данный модуль зависит от модуля ccl-helpers-module этого же проекта. <br>
Но эта зависимость добавляется посредством собранного jar-файла, а не ссылкой на исходники модуля.<br>
Соответственно, для успешной сборки данного модуля jar-архив зависимого модуля должен быть предварительно опубликован в репозитории (локальном или
публичном).<br>
Для использования шагов данного модуля необходимо в .conf-файле в ресурсах проекта указать параметры подключения к БД, и в директории
_**/resources/queries**_ создать файл .sql с именем из поля "queryFilePath", в котором будут храниться именованные запросы к БД (см. Описание модуля).
