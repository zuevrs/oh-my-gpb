# ccl-helpers-module
## Назначение сервиса
Сервис ccl-helpers-module предоставляет утилитные вспомогательные классы и модельные структуры, используемые в других CCL-модулях. Он включает инструменты для работы с файлами (.csv, .docx, .xlsx, .xml, .pdf), строками, числами, списками, датами, генерацию тестовых данных, подключение к Vault-хранилищу, проверки условий и параллельное выполнение задач. Сервис относится к домену утилит для автоматизации тестирования.
## Описание процесса
В сервисе реализован следующий бизнес-процесс: Поддержка вспомогательных функций:
1. Обработка файлов различных форматов и их содержимого.
2. Управление строками, числами, списками и датами с использованием заданных паттернов.
3. Генерация тестовых данных (строки, числа, даты, UUID и т.д.).
4. Подключение к Vault-хранилищу для безопасного хранения данных.
5. Выполнение проверок (assert'ы) для строк, чисел, дат, JSON и булевых значений.
6. Параллельное выполнение методов через IAsyncExecutor.
7. Настройка через .conf-файл с параметрами системных слов и форматов дат.

## Описание плагина ccl-helpers-module
Данный модуль содержит различные утилитные вспомогательные классы, используемые в других ccl-модулях, а также некоторые модельные классы для таблиц
параметров шагов:
- вспомогательные классы для работы с файлами, в том числе файлами форматов .csv, .docx, .xlsx, .xml, .pdf;
- вспомогательные классы для работы со строками (в том числе с html), числами, списками, датами;
- генератор тестовых данных - строк, чисел, дат, uuid и т.п.;
- класс для подключения к Vault-хранилищу;
- классы-assert'ы - содержат проверки строк, чисел, дат, json'ов, булевых значений на соответствие различным условиям;
- модельные классы для таблиц параметров шагов;
- класс IAsyncExecutor для выполнения каких-либо методов параллельно в несколько потоков.
  <br>
  
## Конфигурация

Для работы со строками, датами и подключения к VaultStore с помощью классов данного модуля требуется в .conf-файле вашего проекта добавить
следующие параметры:
```
#Общие настройки
systemWords {
    linux {
        blank = ""
        space = " "
        new_line = "\n"
    }
    windows {
        blank = ""
        space = " "
        new_line = "\r\n"
    }
}

#паттерны для парсинга дат, при отсутствии необходимого паттрена - добавить в список
date.comparison {
    patterns = [
        "yyyy-MM-dd'T'HH:mm[:ss][.SSS][ZZZ]",
        "dd-MM-yyyy[ HH:mm:ss]",
        "dd.MM.yyyy[ HH:mm:ss][.SSS]",
        "yyyy-MM-dd[ HH:mm:ss][.SSS]",
        "dd.MM.yyyy[ HH:mm:ss][.SSSSSS]",
        "yyyy-MM-dd[ HH:mm:ss][.SSSSSS]",
        "dd.MM.yyyy, HH:mm:ss[.SS]"]
}

vault.path.list = ["ccl/application"]
```

## Список шагов модуля
Это утилитный модуль, не содержащий шагов.

## Требования
Данный модуль зависит от модуля akita-gpb-core-module этого же проекта. <br>
Но эта зависимость добавляется посредством собранного jar-файла, а не ссылкой на исходники модуля.<br>
Соответственно, для успешной сборки данного модуля jar-архив зависимого модуля должен быть предварительно опубликован в репозитории (локальном или
публичном).<br>
Также для работы данного модуля необходимо в .conf-файле вашего проекта добавить требуемые параметеры (см. Описание).
