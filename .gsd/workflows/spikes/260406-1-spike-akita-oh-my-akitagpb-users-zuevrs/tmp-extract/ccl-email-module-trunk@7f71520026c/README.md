# ccl-email-module
## Назначение сервиса
Сервис ccl-email-module предназначен для получения и отправки email-писем по различным протоколам. Реализована отправка по SMTP (технический пользователь SRV-CCL-T-EXCHANGE@gazprombank.ru) и получение Outlook-почты по Microsoft-EWS. Сервис относится к домену автоматизации тестирования email-коммуникаций.
## Описание процесса
В сервисе реализован следующий бизнес-процесс: Управление email-коммуникациями:
1. Конфигурация параметров Outlook-почты в файле .conf (host, address, gpbuUser, password, timeouts).
2. Отправка писем по SMTP-протоколу.
3. Получение и проверка входящих сообщений по Microsoft-EWS (ожидание, фильтрация по параметрам).
4. Сохранение тем, тел, вложений и ссылок из писем в переменные.
5. Проверка отсутствия сообщений в течение указанного времени.

## Конфигурация
Для конфигурации параметров Outlook-почты, на которую ожидается получение писем, в .conf-файле проекта необходимо добавить следующие параметры
(и подставить в поля address, xxx и password свои данные):
```
email {
    host = "xxx.xxx.gazprombank.ru"
    address = "xx"
    xxx = "xxx"
    password = "toChange"
    waiting {
        timeout.seconds = 300
        pollInterval.seconds = 5
    }
    lastItemsFetchCount = 20
}
```
Данные шаги будут работать только на том сервере, у которого открыты сетевые проходы до соответствующих почтовых серверов
(xxx.xxx.gazprombank.ru, xxx.xxx.gazprombank.ru).

## Требования
Данный модуль зависит от модуля ccl-helpers-module этого же проекта. <br>
Но эта зависимость добавляется посредством собранного jar-файла, а не ссылкой на исходники модуля.<br>
Соответственно, для успешной сборки данного модуля jar-архив зависимого модуля должен быть предварительно опубликован в репозитории (локальном или
публичном).<br>
