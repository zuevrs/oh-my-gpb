# ccl-camunda-module
## Назначение сервиса
Сервис ccl-camunda-module предназначен для взаимодействия с Camunda через REST-запросы, включая ожидание переходов токенов, проверку порядка кубиков, управление переменными, загрузку/удаление BPMN-процессов и DMN-таблиц, завершение задач, проверку инцидентов и работу с XML-файлами BPMN-схем. Сервис относится к домену автоматизации процессов Camunda.
## Описание процесса
В сервисе реализован следующий бизнес-процесс: Управление Camunda-процессами:
1. Ожидание перехода токена на указанный кубик процесса.
2. Проверка порядка прохождения кубиков и их переменных.
3. Загрузка и удаление BPMN-процессов и DMN-таблиц.
4. Завершение задач и обновление переменных процесса.
5. Выполнение HTTP-запросов к Camunda-API.
6. Управление хуками для логирования и очистки инстансов после тестов.

В модуль входят следующие классы:
- модели REST-запросов и класс с самими запросами;
- модели таблиц параметров шагов;
- хуки;
- и непосредственно сами шаги.
  <br>
  Для работы модуля в .conf-файле вашего проекта требуется добавить пользовательские параметры, поменяв при необходимости значения полей:
```
camunda {
    base.url = "https://ebg-camunda-demo.xxx.xxx.gazprombank.ru/engine-rest"
    #Camunda timeouts (in seconds)
    activityInstance.timeout = 180
    incidents.timeout = 5
    activityInstance.pollInterval = 1
    #Camunda requests repeat count (для шагов ожидания перехода на кубик):
    #Число попыток запросов при нахождении на одном и том же кубике
    sameActivityInstance.repeatCount = 160
    #Число попыток запросов при нахождении на одном и том же процессе
    sameProcessInstance.repeatCount = 140
}
```
Для работы AfterAll-хука, удаляющего после тестов инстансы процессов, созданные в тесте, необходимо в вашем проекте подключить фильтр RestAssured'а:
```
RestAssured.filters(new StartProcessInstanceFilter());
```

## Список хуков модуля
- _**getLogsFromCamunda**_ - @After-хук (выполняется для каждого теста), собирает все переменные и инциденты Camunda, собранную информацию прикрепляет
  к результатам теста в виде файла.<br>
  Выполняется только для упавших тестов, и которые помечены тегом _**@Camunda**_.
- **_deleteStartedProcessInstances_** - @AfterAll-хук (выполняется после всего прогона), удаляет инстансы процессов в Camunda, если старт процесса
  был произведён из теста REST-запросом "${camundaBaseURL}/process-definition/.+/start".<br>
  Если на Сценарий теста добавить тег **@saveProcessInstance**, то все созданные в тесте таким способом инстансы процессов удалены не будут (и все
  задеплоенные процессы соответственно тоже).<br>
  Для работы данного хука необходимо в вашем проекте подключить фильтр RestAssured'а:
```
RestAssured.filters(new StartProcessInstanceFilter());
```
- _**deleteCamundaDeployments**_ - @AfterAll-хук (выполняется после всего прогона), BPMN-процессы и DMN-таблицы в Camunda, если
  что-то было загружено в тестах с помощью специальных шагов (например, _@И("^загрузить BPMN-процессы и/или DMN-таблицы в Camunda")_), а не простым
  REST-запросом.<br>
  По умолчанию удаление деплойментов производится с параметром _"cascade=false"_. Но если на Сценарий теста добавить тег **_@deleteDeploymentCascade_**,
  то будет производиться каскадное удаление (_"cascade=true"_).
  Если на Сценарий теста добавить тег **@saveProcessInstance**, то все задеплоенные в тесте процессы удалены не будут (чтобы сохранить инстансы
  процессов).

## Требования
Данный модуль зависит от модулей ccl-helpers-module, ccl-keycloak-module и akita-gpb-api-module этого же проекта. <br>
Но эти зависимости добавляются посредством собранных jar-файлов, а не ссылкой на исходники модулей.<br>
Соответственно, для успешной сборки данного модуля jar-архивы зависимых модулей должен быть предварительно опубликованы в репозитории (локальном или
публичном).<br>
Также необходимо в вашем проекте в .conf-файле добавить необходимые модулю параметры (см. Описание).

### Зависимости
```
ru.gazprombank.automation:ccl-helpers-module
ru.gazprombank.automation:ccl-keycloak-module
ru.gazprombank.automation:akita-gpb-api-module
```
