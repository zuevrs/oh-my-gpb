# ccl-pre-post-condition-module
## Назначение сервиса
Сервис ccl-pre-post-condition-module предоставляет хуки для реализации предусловий и постусловий, выполняемых перед и после тестов. Он оптимизирует выполнение параметризованных тестов, используя теги @PreCondition и @PostCondition. Сервис относится к домену управления условиями тестирования в автоматизации.
## Описание процесса
В сервисе реализован следующий бизнес-процесс: Управление пред- и постусловиями:
1. Выполнение предусловий из указанного сценария с тегом @PreCondition один раз перед всеми итерациями параметризованного теста (в отличие от стандартного поведения Cucumber).
2. Выполнение постусловий из указанного сценария с тегом @PostCondition после каждого теста независимо от результата.
3. Очистка кэша и инициализация через тег @NoInitBefore для сценариев пред- и постусловий.

## Описание модуля
Данный модуль содержит только хуки предусловий и постусловий, выполняемых перед тестами / после тестов.
1. Тег **_@PreCondition_** - навешивается на Сценарий, применим только к параметризованным тестам (Структура сценария / Scenario outline, имеющих
   блок Примеры / Examples). Выполняет действия из другого Сценария, указанного в теге, ОДИН РАЗ перед прогоном параметризованного теста со всеми
   наборами параметров - в отличие от дефолтного Предусловия Кукумбера, которое выполняется перед каждой итерацией параметризованного теста, для каждого
   набора параметров.
2. Тег **_@PostCondition_** - навешивается на Сценарий. Выполняет действия из другого Сценария, указанного в теге, непосредственно после самого теста
   и независимо от результата прохождения теста (прошёл тест или упал).<br>
   Подробнее написанно [здесь](https://xxx.xxx.gazprombank.ru/pages/viewpage.action?pageId=872154597). <br>

<br>
Все Сценарии предусловий и постусловий, указанные в аннотациях @PreCondition и @PostCondition, должны иметь тег _**@NoInitBefore**_.<br>

## Список шагов модуля
Этот модуль содержит только хуки.

## Список хуков модуля
- _**runPreconditionFeature**_ - @Before-хук (выполняется для каждого теста), запускает Сценарии, указанные во всех @PreCondition-тегах
  исходного теста, ОДИН РАЗ перед запуском всех параметров параметризованного теста (в отличие от дефолтного Предусловия Кукумбера, выполняемого
  перед каждой итерацией параметризованного теста, для каждого набора параметров).<br>
- **_runFeatureInFeature_** - @After-хук (выполняется для каждого теста), запускает Сценарии, указанные во всех @PostCondition-тегах исходного теста,
  после завершения этого теста.<br>
  
## Требования
Данный модуль зависит от модуля ccl-helpers-module этого же проекта. <br>
Но эта зависимость добавляется посредством собранного jar-файла, а не ссылкой на исходники модуля.<br>
Соответственно, для успешной сборки данного модуля jar-архив зависимого модуля должен быть предварительно опубликован в репозитории (локальном или
публичном).<br>
На Сценарии тестов, в которых необходимо выполнять кастомные Предусловия / Постусловия, требуется добавить теги @PreCondition / @PostCondition.<br>
Все Сценарии предусловий и постусловий, указанные в аннотациях @PreCondition и @PostCondition, должны иметь тег _**@NoInitBefore**_.<br>
