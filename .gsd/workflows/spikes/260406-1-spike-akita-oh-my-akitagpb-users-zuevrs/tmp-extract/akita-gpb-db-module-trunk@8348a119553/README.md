# Akita GPB module akita-gpb-db-module

## Назначение сервиса
Сервис akita-gpb-db-module обеспечивает взаимодействие с базами данных через выполнение SQL-запросов и обработку результатов. Он использует HikariCP для управления пулом подключений, Spring Data JDBC для CRUD-операций и поддерживает PostgreSQL, MySQL, MariaDB, Oracle, SQL Server, H2, DB2 и Derby. Модуль интегрируется с Jayway JsonPath для обработки результатов и зависит от модуля akita-gpb-core-module для передачи переменных между шагами. Сервис относится к домену автоматизации тестирования баз данных.
## Описание процесса
В сервисе реализован следующий бизнес-процесс: Управление доступом к базам данных:
1. Создание подключения к базе данных с указанными параметрами (драйвер, URL, логин, пароль).
2. Выполнение SQL-запросов к базе данных.
3. Преобразование и сохранение результатов запросов в переменные.
4. Проверка соответствия значений в ответах из базы данным из таблицы.
5. Управление ограничениями и валидация данных.

## Список шагов модуля
Общие шаги:
```
@И("^создано подключение к БД с драйвером: \"(.*)\", url: \"(.*)\", логином: \"(.*)\" и паролем: \"(.*)\", сохранено в переменную \"(.*)\"$")
@И("^выполнен запрос \"(.*)\" к БД: \"(.*)\". Результат\\(название поля, значение\\) сохранен в переменную \"(.*)\"")
@И("^полученный ответ \"(.*)\" от БД преобразован в список значений и сохранен в переменную \"(.*)\"$")
@И("^получено количество полей в ответе от БД \"(.*)\" и сохранено в переменную \"(.*)\"$")
@И("^в ответе от БД \"(.*)\", найденные значения равны значениям из таблицы$")
@И("^из ответа от БД \"(.*)\" значение столбца \"(.*)\" сохранено в переменную \"(.*)\"$")
@И("^значения из ответа \"(.*)\" попадают под следующие ограничения$")

```

## Требования
Данный модуль зависит от модуля akita-gpb-core-module этого же проекта. <br>
Но эти зависимости добавляются посредством собранных jar-файлов, а не ссылкой на исходники модулей.<br>
Соответственно, для успешной сборки данного модуля jar-архивы зависимых модулей должен быть предварительно опубликованы в репозитории (локальном или
публичном).

Дополнительные настройки:

Для работы с БД в файле build.gradle необходимо добавить зависимость – драйвер. Пример для Sybase:

implementation group: 'com.sybase.jdbc', name: 'jconn4', version: 4

implementation group: 'com.oracle', name: 'ojdbc14', version: '10.2.0.4.0'
implementation group: 'org.postgresql', name: 'postgresql', version: '42.6.0'
implementation group: 'com.microsoft.sqlserver', name: 'mssql-jdbc', version: '12.3.1.jre11-preview'

*актуальную версию смотреть в нексус
